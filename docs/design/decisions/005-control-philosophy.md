# 制御思想

## 概要

teleop-hand の制御思想を決定する。指間距離から関節角度への変換方法（逆運動学）、制御則の構造、力フィードバック方式を定める。

## 設計判断

### 判断1: 逆運動学 — 余弦定理による角度変換

**問題**: finger-tracker が計測した指間距離 (mm) をモータの目標角度 (rad) にどう変換するか

**選択肢**:
1. 線形マッピング（指間距離 → 目標角度の線形変換）
2. 余弦定理による逆運動学（リンク機構の幾何学を考慮）
3. ルックアップテーブル（実測値ベースの変換テーブル）

**決定**: 余弦定理による逆運動学

**理由**:
- ロボットハンドの指機構は等長リンク（r = 0.0725 m）で構成されており、余弦定理で指間距離から関節角度を解析的に求められる
- 線形マッピングではリンク機構の非線形性を無視するため、開き量の大きい領域で誤差が増大する
- ルックアップテーブルは実測が必要で手間がかかる

**変換式**:

```
x_d: 指間距離 [m]（finger-tracker の distance_mm を m に変換）
r:   リンク長 = 0.0725 [m]
θ_cmd: 目標関節角度 [rad]

余弦定理: x_d² = r² + r² - 2r²cos(θ_cmd)
         x_d² = 2r²(1 - cos(θ_cmd))

θ_cmd = cos⁻¹(1 - x_d² / (2r²))

有効範囲: 0 ≤ x_d ≤ 2r = 0.145 m (145 mm)
x_d が範囲外の場合はクランプ: x_d = clamp(x_d, 0.0, 2*r)
```

**幾何学的解釈**:

```
        θ_cmd
       ╱     ╲
    r ╱       ╲ r
     ╱         ╲
    ●───────────●
         x_d
         (指間距離)

r = 0.0725 [m]（リンク長、実機計測値）
```

2 本の等長リンク（r）が角度 θ_cmd で開き、その先端間距離が x_d に対応する。

**パラメータ**:

| パラメータ | 値 | 説明 | 設定場所 |
|-----------|-----|------|---------|
| r (link_length) | 0.0725 | リンク長 [m] | config JSON |

**トレードオフ**:
- **利点**: 機構の非線形性を正確に反映、解析的な閉形式解のため計算コストが低い
- **欠点**: 2 リンク等長の仮定に依存。機構が変更された場合は式の見直しが必要

### 判断2: 制御則 — PD 制御 + DOB 補償

**問題**: remote モードの制御則をどのような構造にするか

**決定**: PD 位置制御 + 外乱オブザーバ (DOB) 補償

**制御則**:

```
// 逆運動学（判断1）
θ_cmd = cos⁻¹(1 - x_d² / (2r²))

// PD 位置制御 + DOB
θ̈_f = (C_p + s·C_d)(θ_cmd - θ_f) + f_dis

ここで:
  θ_f:   モータ応答角度 [rad]（エンコーダ計測値）
  θ̈_f:  モータ角加速度指令 [rad/s²]
  C_p:   位置ゲイン（= k_p）
  C_d:   速度ゲイン（= k_v）
  f_dis: DOB 推定外乱
  s:     微分演算子

// トルク指令への変換: f_ref = M * θ̈_f（ロータ慣性 M をかけてトルク次元に変換）

// 指令値フィルタ（判断3）
θ_cmd_filtered = LPF(θ_cmd, g_cmd)   // g_cmd = 60 rad/s

// 実装形式（controller_try 準拠）
f_ref = M * (k_p * (θ_cmd_filtered - θ_res) + k_v * (0 - dθ_res))
f_vol = f_ref + f_dis
f_out = f_vol
```

**理由**:
- controller_try の位置制御構造をそのまま適用可能
- DOB により摩擦・重力等の外乱を補償し、安定した位置追従を実現
- パラメータ（k_p, k_v, g_dis）は 4018_finger.json の調整済み値を流用

**トレードオフ**:
- **利点**: 実績あるパラメータを流用、DOB による外乱補償
- **欠点**: 速度指令は 0（dθ_cmd = 0）としているため、指令値の急変時に追従遅れが生じうる

### 判断3: 指令値フィルタ — 1次遅れフィルタ（g_cmd = 60 rad/s）

**問題**: finger-tracker の指令値更新（30Hz）と制御ループ（デフォルト 10kHz、[ADR 004](./004-control-architecture.md) で設定可能）の周波数差をどう扱うか

finger-tracker は 30fps で指令値を送信するが、制御ループはデフォルト 10kHz で実行される。指令値は 33ms ごとに階段状にジャンプし、PD 制御器にステップ入力が加わるため、カクつきや振動が発生しうる。

```
指令更新(30Hz):  |_______________|_______________|________
制御ループ(10kHz): __|__|__|__|__|__|__|__|__|__|__|__|__|__
                     333回同じ値        次の値（ジャンプ）
```

**選択肢**:
1. フィルタなし（ゼロ次ホールド: 受信値をそのまま保持）
2. 1次遅れフィルタ（指令値をLPFで平滑化）
3. 線形補間（前回値と今回値の間を補間）

**決定**: 1次遅れフィルタ（g_cmd = 60 rad/s）

**理由**:
- controller_try の擬似微分・DOB と同じフィルタ構造で実装が統一される
- 階段状のジャンプを滑らかに変換し、カクつきを防ぐ
- g_cmd = 60 rad/s（≈ 10Hz）は指令更新 30Hz に対して十分低く、平滑化効果がある
- 線形補間は到着タイミングの管理が複雑になる

**伝達関数**:

```
            g_cmd
G(s) = ─────────────
          s + g_cmd

g_cmd = 60 [rad/s]（≈ 10 Hz）
```

**離散実装**（controller_try の signal_processing.h と同じ構造）:

```cpp
// 毎制御周期（10kHz, dt = 0.0001s）で実行
buf_cmd += dt * g_cmd * (theta_cmd_raw - buf_cmd);
theta_cmd_filtered = buf_cmd;
```

**信号の流れ**:

```
受信(30Hz)                  フィルタ(10kHz)              制御則(10kHz)
θ_cmd_raw ──→ 1次遅れLPF ──→ θ_cmd_filtered ──→ PD + DOB ──→ f_out
(階段状)       g_cmd=60        (滑らか)
```

**パラメータ**:

| パラメータ | 値 | 説明 | 設定場所 |
|-----------|-----|------|---------|
| g_cmd | 60.0 | 指令値フィルタカットオフ [rad/s] | config JSON |

**調整の目安**:
- g_cmd を大きくする → 応答が速いがカクつき残る
- g_cmd を小さくする → 滑らかだが応答が遅れる
- 30〜100 rad/s の範囲で調整。まず 60 rad/s から

**トレードオフ**:
- **利点**: 実装が簡単（1行）、controller_try のフィルタ構造と統一、カクつき防止
- **欠点**: フィルタ分の位相遅れ（g_cmd=60 で約 17ms）。操作者の手の動きに対して微小な遅延が生じる
