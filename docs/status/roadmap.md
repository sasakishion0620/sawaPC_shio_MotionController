# ロードマップ

## Phase 1: 基盤構築

目標: controller_try のコードを整理・移植し、単体でモータ制御ができる状態にする

- config モジュール（関節パラメータ・システム設定の読み込み）
- hardware モジュール（Contec DA/カウンタの移植）
- controller モジュール（基本制御モードの移植: idle, DA_check）
- record_motion スレッド（データ記録、[ADR 004](../design/decisions/004-control-architecture.md)）
- gui モジュール（制御モード切替UI）
- CMake ビルド環境の構築

## Phase 2: 通信・遠隔操作

目標: finger-tracker の指間距離を受信し、ロボットハンドを遠隔操作できるようにする

- comm モジュール（通信方式の決定・実装）
- finger-tracker 側の送信インターフェース実装
- 指間距離 → モータ指令値への変換ロジック
- エンドツーエンドでの遠隔操作動作確認

## Phase 3: バイラテラルテレオペレーション

目標: 力フィードバックを含むバイラテラルテレオペレーションを実現する

- 力センシング/推定の実装
- バイラテラル制御アルゴリズムの実装
- マスター-スレーブ間の力・位置双方向伝達
- 安定性・透明性の評価
