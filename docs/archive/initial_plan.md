# teleop-hand 初期仕様書

## プロジェクト概要

finger-tracker が計測した指間距離（mm）を指令値として受信し、ロボットハンドの把持動作をリアルタイムに遠隔操作するシステム。既存の controller_try（C++ リアルタイムモーション制御フレームワーク）をベースに、通信層・制御ロジック・ハードウェアI/O を整理・改修して構築する。

## 動機

既存研究で、1自由度ロボットハンドによる柔軟物体把持を位置・力ハイブリッド制御で実現している。finger-tracker プロジェクトでは人間の指の開閉量（指間距離）をリアルタイムに計測する仕組みを構築した。本プロジェクトはその計測値をロボットハンドの遠隔操作指令値として使用し、将来的には反力を操作者に返すバイラテラルテレオペレーションシステムへの発展を目指す。

## コア機能

1. **指令値受信**: finger-tracker からの指間距離（mm）をリアルタイムに受信する
2. **制御モード切替**: GUI から制御モード（idle、位置制御等）を切り替える
3. **モータ制御**: 指令値に基づいてロボットハンドのモータを制御する
4. **エンコーダ読み取り**: カウンタボードからモータの回転角度を取得する
5. **リアルタイムGUI**: 関節状態・制御モードをリアルタイムで表示する

## アーキテクチャ

```
finger-tracker（別プロセス）
    │ 指間距離 (mm)
    ▼
comm（通信層）── 指令値受信
    │
    ▼
controller（制御ロジック）── モード切替・制御計算
    │
    ▼
hardware（ハードウェアI/O）
    ├── DA ボード → モータ電圧出力
    └── カウンタ ← エンコーダパルス読み取り
```

### モジュール構成

| モジュール | 責務 |
|-----------|------|
| comm | finger-tracker との通信（指令値受信） |
| controller | 制御ロジック（モード切替・制御計算） |
| hardware | ハードウェアI/O（Contec DA/カウンタ） |
| config | 設定管理（JSON） |
| gui | GUI表示（OpenGL） |
| scripts | ユーティリティスクリプト |

## 技術要件

### ハードウェア

| 区分 | 機材 | 備考 |
|------|------|------|
| ロボットハンド | 研究室のロボットハンド | 詳細は今後確認 |
| モータ | サーボモータ（エンコーダ付き） | 関節パラメータは config/joints/ で管理 |
| DAボード | Contec PCI DAボード（16bit） | モータへの電圧出力 |
| カウンタボード | Contec PCI カウンタボード | エンコーダパルス読み取り |
| PC | Linux PC（PCI スロット搭載） | controller_try と同じ環境 |

### ソフトウェア

| 区分 | 内容 |
|------|------|
| コーディング | VSCode (WSL2) |
| 実行環境 | Linux PC（root権限、PCIアクセス） |
| 言語 | C++ |
| ビルド | CMake |
| 主要ライブラリ | Eigen, Boost.PropertyTree, OpenGL, GLFW, GLEW |
