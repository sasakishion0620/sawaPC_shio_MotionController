# ユースケースガイド

## 目的

teleop-hand の操作手順をユーザー視点で記録する。

### 対象読者

- システムの操作者（研究者）

## ディレクトリ構成

```
usecases/
├── GUIDE.md           # 本ガイド
├── TEMPLATE.md        # ユースケーステンプレート
└── {usecase}.md       # 個別ユースケース
```

## 種別

| 種別 | 説明 | 例 |
|------|------|---|
| セットアップ | 環境構築・ハードウェア接続 | Contec ボードの有効化 |
| 遠隔操作 | テレオペレーションの実行 | finger-tracker 連携での把持操作 |
| キャリブレーション | 調整手順 | モータのゲイン調整 |

## セクション構成

| セクション | 必須/推奨 |
|-----------|----------|
| 概要 | 必須 |
| 前提条件 | 必須 |
| 操作手順 | 必須 |
| 期待結果 | 必須 |
| トラブルシューティング | 推奨 |

## 記法規約

### 操作フロー

ステップ番号 + 期待結果の形式で記述:

```markdown
1. Contec ボードを有効化する
   ```bash
   sudo ./scripts/enable_contec_board.sh
   ```
   - **期待結果**: PCI デバイスが認識される

2. 制御プログラムを起動する
   ```bash
   sudo ./build/control
   ```
   - **期待結果**: GUI が表示され idle モードで待機する
```

## テンプレートの使用

`TEMPLATE.md` をコピーし、プレースホルダを埋める。ファイル名: `{kebab-case-usecase}.md`

## 品質チェックリスト

- [ ] 操作手順が順序通りに記述されている
- [ ] 各ステップに期待結果がある
- [ ] 前提条件（ハードウェア・ソフトウェア）が明記されている
- [ ] コマンドがコピーペースト可能
